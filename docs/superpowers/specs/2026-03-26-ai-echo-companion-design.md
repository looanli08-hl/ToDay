# AI Echo 智能陪伴系统设计

**日期**: 2026-03-26
**状态**: 已确认，待实现

---

## 1. 产品定位

Echo 不是一个功能，是一个**角色** — 一个住在 App 里、通过用户的生活数据真正了解用户的知音。

**核心能力：**
- **洞察者** — 通过日常数据理解用户是什么样的人，像镜子一样准确描述
- **知音** — 有温度地关心，知道什么时候该说什么
- **整理者** — 识别用户随口说的重要内容，归类整理，合适时提醒
- **不遗忘** — 替用户记住灵光一现，在对的时刻唤醒

**核心原则：**
- 这个应用全都是为用户个人打造的 — 让用户信任，分析陪伴才更准确
- AI 辅助但不替代用户思考 — 主观权归用户

## 2. AI 服务分层

| 层级 | 模型 | 适用场景 | 成本 |
|------|------|---------|------|
| **免费层** | Apple Foundation Models（本地） | 日记总结、情绪分类、简短 Echo 回应 | 免费 |
| **Pro 层** | DeepSeek API（或用户自选） | 深度 Echo 对话、个性化关怀、AI 日记生成 | 订阅 |
| **开源层** | 用户自行接入任意 API | 自定义模型 | 用户自理 |

## 3. 四层记忆架构

```
层级 1：用户画像（长期记忆）
  ├── AI 生成的用户总结（"这个人是什么样的"）
  ├── 性格特征、生活习惯、作息规律
  ├── 每周更新一次，压缩成几百字
  └── 每次对话都带上作为 context

层级 2：近期动态（短期记忆）
  ├── 最近 7 天的每日摘要
  ├── 近期情绪变化趋势
  ├── 未回看的快门记录
  ├── 每天睡前自动总结
  └── 强烈情绪内容立即触发更新

层级 3：当日数据（工作记忆）
  ├── 今天的完整时间线
  ├── 今天的快门/心情记录原文
  └── 实时

层级 4：对话记忆
  └── Echo 和用户的历史对话摘要
  └── 跟随画像一起更新
```

**数据范围：** Echo 能看到全部数据 — 快门、心情、运动、睡眠、屏幕时间、时间线事件、历史趋势。

**AI 每次收到的 context：**
```
画像（~200字）+ 近期摘要（~300字）+ 对话记忆（~200字）+ 当日数据（具体内容）
```

**更新频率：**
- 用户画像：每周
- 近期动态：每天（睡前自动）+ 强情绪即时
- 当日数据：实时
- 对话记忆：每次对话后更新

## 4. Echo 人格系统

**默认人格模板：**

| 人格 | 风格 | System Prompt 关键词 |
|------|------|---------------------|
| **温柔内敛** | 安静的老朋友，不多话但句句到点上 | 温和、简洁、深思、不主动给建议除非被问到 |
| **积极阳光** | 贴心的闺蜜/兄弟，热情鼓励 | 热情、鼓励、emoji 适量、主动分享感受 |
| **克制理性** | 成熟的导师，客观分析 | 理性、条理清晰、数据驱动、少情绪化表达 |

用户可以后续自定义人格描述。

## 5. Echo 交互场景

### 5.1 用户主动对话

Echo 页面有对话入口，用户随时可以跟 Echo 聊：
- 「你觉得我最近状态怎么样？」
- 「帮我整理一下这周的想法」
- 「我今天心情不好」
- 自由对话

### 5.2 系统自动生成

**每日洞察（每天生成一次）：**
- 基于当天数据生成一段简短的 Echo 卡片
- 例：「今天运动了 45 分钟，比昨天多了不少。晚上早点休息吧。」

**智能回响（替代原有的定时提醒）：**
- 用户 3 天前说了「突然想到一个产品创意」
- Echo 不是机械提醒，而是：「你 3 天前提到一个产品创意，想聊聊吗？有时候隔几天再看会有新的角度。」

**想法整理：**
- 用户这周零散记了 5 条相关的快门想法
- Echo 归类总结：「你最近一直在想 XX 这件事，要不要我帮你整理一下？」

**待办识别：**
- 用户快门里说「下周要交论文」
- Echo 识别为时间敏感内容，到期前提醒

### 5.3 推送分寸感

- 用户可自行设定推送频率和时段
- 后续接入手表后更方便推送查看
- 用户连续忽略后自动降频

## 6. 特色功能

### 6.1 「我眼中的你」— 镜子功能

- Echo 页面一个小入口：「Echo 眼中的你」
- 点击后 AI 基于画像生成一段对用户的描述
- 用户不满意可以随时跟 AI 说，AI 会调整画像
- 适合截图分享 — 天然传播点

### 6.2 临时会话模式

- 类似 AI 大模型的临时会话
- 用户可以选择「这次对话不记入记忆」
- 保护隐私同时保留功能

---

## 7. 冷启动策略

| 时间 | Echo 状态 |
|------|----------|
| 第 1 天 | 功能可用但 Echo 坦诚「我还在了解你」。做轻度互动（鼓励记录） |
| 第 2-3 天 | 开始能评论当天的数据（「今天走了不少路」） |
| 第 3-5 天 | 生成初步画像，Echo 的回应开始个性化 |
| 第 7 天+ | 完整画像建立，Echo 真正「认识」用户 |

## 8. 技术架构

```
┌─────────────────────────────────────────┐
│              Echo UI Layer               │
│  对话界面 / 洞察卡片 / 镜子功能 / 设置     │
├─────────────────────────────────────────┤
│            EchoAIService                 │
│  统一接口，根据用户层级路由到不同模型        │
│  ├── AppleLocalProvider (免费)            │
│  └── DeepSeekProvider (Pro)              │
├─────────────────────────────────────────┤
│           EchoMemoryManager              │
│  管理四层记忆的读写和更新                    │
│  ├── UserProfile (画像)                   │
│  ├── RecentSummary (近期摘要)             │
│  ├── DailyData (当日数据收集)              │
│  └── ConversationMemory (对话记忆)        │
├─────────────────────────────────────────┤
│           EchoPromptBuilder              │
│  组装 context: 画像+近期+当日+人格模板      │
├─────────────────────────────────────────┤
│          EchoScheduler                   │
│  管理自动生成的时机和频率                    │
│  ├── 每日洞察生成                          │
│  ├── 智能回响触发                          │
│  ├── 想法整理检测                          │
│  └── 待办识别                             │
├─────────────────────────────────────────┤
│        现有数据层 (已完成)                  │
│  ShutterManager / MoodRecord /           │
│  DayTimeline / HealthKit                 │
└─────────────────────────────────────────┘
```

### 关键模块职责

**EchoAIService** — Protocol 定义统一接口：
```
protocol EchoAIProviding {
    func respond(prompt: String, context: EchoContext) async throws -> String
    func summarize(data: DayData) async throws -> String
    func generateProfile(summaries: [DailySummary]) async throws -> String
}
```

**EchoMemoryManager** — SwiftData 持久化四层记忆：
- `UserProfileEntity` — 画像文本 + 最后更新时间
- `DailySummaryEntity` — 每天一条摘要
- `ConversationEntity` — 对话历史摘要

**EchoPromptBuilder** — 组装最终发给 AI 的 prompt：
- 人格模板（system prompt 前缀）
- 用户画像
- 近期 7 天摘要
- 当日数据
- 对话记忆
- 用户的当前输入

**EchoScheduler** — 决定什么时候自动触发 AI：
- 每日睡前生成总结
- 检测快门中的时间敏感内容
- 检测零散想法是否可以归类
- 控制推送频率

## 9. 不在 MVP 范围内

- 语音对话（Echo 说话）
- 手表端 Echo 交互
- 多用户/家庭共享
- 第三方 AI 模型市场
