# Echo 消息中心设计

**日期**: 2026-03-26
**状态**: 待确认

---

## 1. 产品定位

Echo tab 从「单一对话页」升级为**消息中心** — Echo 像微信好友一样主动给用户发消息，每条消息都有明确的来源和原因。

**核心体验**：打开 Echo tab，看到 Echo 给你发的消息列表，就像打开微信看到朋友的消息。

---

## 2. 消息类型

| 类型 | 触发条件 | 来源标注 | 示例 |
|------|---------|---------|------|
| **每日洞察** | 每天生成一次 | 来自：今日数据 | 「今天运动了 45 分钟，比昨天多不少」 |
| **快门回响** | 快门记录 N 天后 | 来自：M月D日的快门 | 「你 3 天前提到一个创意，想聊聊吗？」 |
| **想法整理** | 多条快门涉及同一话题 | 来自：最近 N 条快门 | 「你最近一直在想换工作的事」 |
| **情绪关怀** | 检测到情绪变化 | 来自：近期心情趋势 | 「这两天看起来有点累，要不要聊聊？」 |
| **待办提醒** | 快门中有时间敏感内容 | 来自：M月D日的快门 | 「你说下周要交论文，还有 2 天」 |
| **镜子更新** | 画像有新发现 | 来自：你的生活画像 | 「我对你有了新的了解」 |

### 消息卡片结构

每条消息包含：
- **图标** — 类型对应的 emoji/icon
- **标题** — 消息主题
- **预览** — 内容前两行
- **来源标注** — 「来自：XXX」明确说明这条消息的触发原因
- **时间** — 生成时间
- **未读标记** — 未点击过的消息有标记

### 底部入口

列表最底部有一个「跟 Echo 随便聊聊」的入口 — 这是自由对话，不关联任何特定来源。

## 4. 对话详情页

点击消息列表中的一条 → 进入独立对话线程。

### 顶部来源栏
```
← 返回    关于：3月23日快门「产品创意」
```

### 对话内容
- Echo 的第一条消息就是列表里看到的那条（展开完整版）
- 用户可以回复，形成对话
- 对话围绕这个来源主题展开
- Echo 的回复会参考来源数据 + 用户画像

### 对话 context
每个对话线程的 AI context 包含：
```
用户画像 + 近期摘要 + 来源数据（具体的快门记录/时间段/趋势）+ 本线程的对话历史
```

## 5. 数据模型

### EchoMessage（消息列表项）

```
EchoMessage
├── id: UUID
├── type: EchoMessageType (dailyInsight/shutterEcho/thoughtOrg/emotionCare/todoReminder/mirrorUpdate/freeChat)
├── title: String
├── preview: String
├── sourceDescription: String (来源标注文本)
├── sourceData: EchoSourceData? (关联的具体数据)
├── createdAt: Date
├── isRead: Bool
└── threadId: UUID (对应的对话线程)
```

### EchoSourceData（来源数据）

```
EchoSourceData
├── type: SourceType (shutterRecord/dateRange/moodTrend/userProfile)
├── shutterRecordIds: [UUID]? (关联的快门记录 ID)
├── dateRange: DateInterval? (关联的时间段)
└── description: String
```

## 6. 未读红点

- Echo tab 图标上显示未读消息数量
- 点击消息后标记为已读
- 在 AppRootScreen 通过 badge 显示

---

## 7. 与现有系统的关系

- 替换当前的 EchoChatScreen（对话页面）
- 保留 EchoChatViewModel 的 AI 对话能力，但包装在消息线程里
- 保留 EchoScheduler 的调度逻辑，但产出变成消息列表项而不是通知
- 镜子功能变成一种消息类型
- 每日洞察变成一种消息类型

---

## 8. 不在 MVP 范围内

- 消息搜索
- 消息置顶/收藏
- 多个 AI 角色（只有一个 Echo）
- 消息中的图片/语音播放
