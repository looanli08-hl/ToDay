# ToDay 商业化草案 v0.1

## 1. 产品定位

- 产品名：ToDay
- 一句话定位：用 HealthKit 和手动记录，把“今天为什么会这样”整理成一条可以回看的私人时间线。
- 当前形态：iPhone 本地优先应用，先不依赖后端。
- 商业化目标：先验证用户是否愿意为“更清晰地理解自己的一天”付费，再逐步扩展到连续洞察和长期报告。

## 2. 目标用户

- 核心用户：已经在意作息、精力、情绪和专注状态的人。
- 第一批最可能付费的人：
  - 使用 Apple Watch / iPhone 健康数据的人
  - 有轻量自我追踪习惯的人
  - 希望更快理解“今天状态为什么差 / 为什么好”的人
- 暂不优先的人：
  - 只想看传统图表步数面板的人
  - 对记录和复盘完全没有习惯的人

## 3. 核心价值承诺

- 不是把更多数据堆给用户。
- 而是把今天的状态、活动、情绪和关键片段，整理成一条可理解的时间线。
- 用户购买的不是“记录工具”，而是：
  - 更快看懂今天
  - 更容易发现自己的状态模式
  - 更容易持续复盘

## 4. 建议的免费 / 付费分层

### 免费层

- 今日时间线基础查看
- 手动快速记录情绪 / 备注
- 当日基础统计
- 最近 3 天回看

### Pro 订阅层

- 7 天 / 30 天 / 月度连续洞察
- 自动生成“今天摘要”
- 状态模式识别：睡眠、活动、情绪、专注之间的关联提示
- 更完整的历史回看
- 自定义标签、复盘模板
- 后续多设备同步与备份

## 5. 第一阶段可收费功能建议

最先建议卖的不是“更多记录入口”，而是“自动洞察”。

优先级建议：

1. 今日摘要生成
2. 近 7 天模式总结
3. 情绪 / 精力波动提示
4. 历史回看限制与 Pro 解锁

原因：

- 用户更容易为结果付费，而不是为输入动作付费。
- “自动理解今天”比“让我再多填一个表单”更接近真正的付费点。

## 6. 建议定价

先用简单订阅验证，不做复杂套餐。

- 月订阅：￥18
- 年订阅：￥128

说明：

- 这个区间适合先做 MVP 验证，不是最终价格。
- 年付需要明显更划算，帮助拉高现金流和留存。
- 如果后续用户明显偏“轻工具”，可以改成更低价位；如果偏“持续洞察”，可以把年价往上抬。

## 7. 关键转化路径

建议的第一版转化路径：

1. 用户首次打开
2. 完成 HealthKit 授权或手动快速记录
3. 当天看到第一条完整时间线
4. 在用户想看“最近 7 天趋势”或“自动总结”时触发付费墙
5. 订阅后立即看到总结结果，而不是只看到一张价格页

付费触发点建议：

- 尝试查看 7 天趋势时
- 想查看自动生成的今日总结时
- 当用户已连续使用 3 天后

## 8. 第一阶段必须补齐的上架资料

- App 对外名称
- 副标题
- 简短卖点文案
- 支持邮箱
- 隐私政策 URL
- 服务条款 URL
- 应用图标
- App Store 截图
- 订阅名称、价格和权益描述

## 9. 法律和合规重点

- 明确说明使用了 HealthKit 数据
- 明确说明数据默认本地优先，不会在未说明情况下上传
- 隐私政策里要写清楚：
  - 收集什么数据
  - 数据存在哪里
  - 是否用于分析
  - 是否与第三方共享
- 不要把产品描述成医疗诊断工具

## 10. 默认品牌文案草案

- 中文名候选：ToDay / 今日线 / 今日感 / 一日线
- 英文副标题候选：Your private timeline for how today actually felt.
- 中文短描述候选：把步数、睡眠、活动和情绪整理成一条真正能看懂的今天。
