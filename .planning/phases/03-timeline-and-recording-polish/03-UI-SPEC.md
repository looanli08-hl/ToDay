# Phase 3 — UI Design Contract
## Timeline and Recording Polish

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher.
> Design goal: "Opening the app at 11pm and seeing your day feels worth doing again tomorrow."
> Source: .impeccable.md, TodayTheme.swift, AppColor/AppFont/AppSpacing tokens, existing feature source files.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SwiftUI native, no shadcn) |
| Preset | not applicable |
| Component library | SwiftUI + UIKit bridging |
| Icon library | SF Symbols (system icons only) |
| Font | SF Pro (system font via `.system(size:weight:design:)`) |

> No third-party UI packages. Zero external Swift Package Manager dependencies per CLAUDE.md. All tokens live in `TodayTheme.swift` / `AppColor` / `AppFont` / `AppSpacing` / `AppRadius`. New code must use `AppColor.*`, `AppFont.*`, `AppSpacing.*`, `AppRadius.*` — never raw values.

## Spacing Scale

Declared values (4pt grid — tokens already in `AppSpacing`):

| Token | Value | SwiftUI Usage |
|-------|-------|---------------|
| `AppSpacing.xxxs` | 2pt | Hairline offsets, sub-pixel corrections |
| `AppSpacing.xxs` | 4pt | Icon-to-label gap, timeline dot spacing |
| `AppSpacing.xs` | 8pt | Compact element spacing, badge padding vertical |
| `AppSpacing.sm` | 12pt | Card internal row gap, chip padding horizontal |
| `AppSpacing.md` | 16pt | Default card padding, section horizontal margin |
| `AppSpacing.lg` | 24pt | Section vertical gap, sheet section spacing |
| `AppSpacing.xl` | 32pt | Major layout gaps between cards |
| `AppSpacing.xxl` | 48pt | Screen top padding, modal hero area |

Exceptions:
- Timeline left column (time labels): fixed 44pt width — preserves HH:mm monospaced alignment
- Timeline connector line: fixed 20pt frame width (8pt for dot, 12pt breathing room)
- EventCard color accent bar: fixed 4pt wide (semantic weight indicator, not a spacing unit)
- Current-time needle: 6pt dot radius, 1.5pt line height
- Bottom action bar corner radius: 20pt (uses `.continuous` style — not in AppRadius tokens, use literal `20`)
- Touch target minimum: 44pt × 44pt (iOS HIG; applies to all tappable timeline rows, mood chips, and action buttons)
- Mood grid: 3-column `LazyVGrid` with 12pt spacing between columns and rows

## Color

> Source of truth: `TodayTheme.swift` / `AppColor`. Hex values below are extracted from UIColor initializers in TodayTheme.swift.

### Surfaces

| Role | Light Mode Hex | Dark Mode Hex | Token |
|------|---------------|---------------|-------|
| Dominant background (60%) | `#F7F5F0` (≈ warm cream) | `#121213` (warm dark) | `AppColor.background` |
| Secondary surface (30%) | `#FFFDF7` (warm white) | `#1C1C1E` | `AppColor.surface` |
| Elevated surface | `#F9F7F2` | `#272728` | `AppColor.surfaceElevated` |
| Separator / border | iOS system separator | iOS system separator | `AppColor.separator` |
| Glass / translucent fill | `#FFFFFF` at 18% opacity | `#FFFFFF` at 18% opacity | `TodayTheme.glass` |

Rule: No pure `#000000`, no pure `#808080`. Every neutral must carry a warm tint. Use `Color(UIColor.systemGroupedBackground)` and `Color(UIColor.secondarySystemGroupedBackground)` for iOS adaptive surfaces — they inherit warm bias from the established `AppColor.background` (which overrides nothing; these are layered above it).

### Accent (10%)

| Token | Hex | Reserved for |
|-------|-----|-------------|
| `AppColor.accent` / `Color.accentColor` | `#5B9A8B` light / `#7CC1AF` dark (teal) | Primary CTA buttons, selected state in date strip and mood grid, active record indicator, "进行中" live badge, timeline current-time dot |

Accent is reserved for exactly these elements. Never apply it to body text, decorative borders, or informational labels.

Second semantic accent: `AppColor.echo` (`#527957` approx `rgb(0.32, 0.58, 0.52)`) — used exclusively for Echo AI callouts and teal-toned section backgrounds.

### Shadows

All shadows use warm-tinted brown base: `Color(red: 0.4, green: 0.3, blue: 0.2)`.

| Level | Token | Radius | Opacity | Y offset |
|-------|-------|--------|---------|----------|
| Subtle | `AppShadow.subtle` | 8pt | 6% | 2pt |
| Elevated | `AppShadow.elevated` | 16pt | 10% | 4pt |

Apply `.appShadow(.subtle)` to cards, action bars, and metric tiles. Apply `.appShadow(.elevated)` to floating sheets and the bottom action bar. Never use `Color.primary.opacity(N)` as shadow color — always use warm tint.

## Component Inventory

Components already implemented that Phase 3 must maintain and refine:

### `DayVerticalTimelineContent` (DayScrollView.swift)
- **Purpose:** Full-day vertical scroll canvas — the product's visual core
- **Upgrade contract for this phase:** Time-of-day gradient background is already implemented. Phase 3 must ensure event row heights are proportional to duration (currently fixed at 76pt/92pt for sleep). Minimum visible row: 44pt touch target.
- **Left column:** 44pt wide, HH:mm in 11pt monospaced medium at 60% opacity (start time) + 10pt at 35% (end time)
- **Connector:** 20pt wide frame. 8pt dot filled with `event.cardFill` at top. 1.5pt spine line filled with `AppColor.label.opacity(0.1)`
- **Gap rows:** Dashed border (`[7, 5]` dash pattern), `opacity(0.35)` separator, "点击记录" capsule affordance bottom-trailing
- **Data gap rows:** "这段时间没有记录 · Xm" in 12pt monospaced quaternary — no tap target needed
- **Current time needle:** 6pt teal dot + 1.5pt separator-colored line, `allowsHitTesting(false)`
- **Mood rows:** 7pt circle (cardStroke fill), capsule layout, 38pt min height

### `EventCardView` (EventCardView.swift)
- **Purpose:** Individual event presentation within the timeline
- **Color accent bar:** 4pt wide, `AppRadius.sm`-ish corner radius (2pt), `event.cardFill`
- **Event name:** 15pt semibold primary
- **Kind badge:** 10pt bold monospaced in `event.cardFill` color
- **Duration:** 13pt bold monospaced tertiary, trailing
- **Detail line:** 13pt regular tertiary — place name + weather + metrics joined by ` · `
- **Background:** `Color(UIColor.secondarySystemGroupedBackground).opacity(0.82)` + `.ultraThinMaterial` layered
- **Blank candidate:** opacity 0.62, dashed border

### `EventDetailView` (EventDetailView.swift)
- **Purpose:** Tap-into sheet for a single event
- **Header card:** `event.cardFill.opacity(0.92)` background. White text on colored backgrounds; `.primary` text on `.quietTime`

- **Sleep stage bar:** 28pt height, per-stage color segments

### `QuickRecordSheet` (QuickRecordSheet.swift)

- **Mood grid:** 3-column `LazyVGrid`, 12pt spacing. Selected state: `Color.accentColor.opacity(0.12)` background + 2pt `Color.accentColor` border

- **Photo thumbnails:** 88pt × 88pt, corner 18pt
- **Action bar:** Sticky `safeAreaInset(.bottom)`, systemGroupedBackground at 96% opacity
- **Detents:** `.medium` and `.large`; drag indicator visible

## Interaction Contracts

### Timeline Event Tap (TML-03)
- Trigger: `.onTapGesture` on `EventCardView`
- Response: Present `EventDetailView` as `.sheet`
- Animation: iOS default sheet presentation (spring-driven from bottom)
- State: `@State private var selectedEvent: InferredEvent?` on `TodayScreen`

### Blank Period Tap — Annotate (TML-04)
- Trigger: `.onTapGesture` on `quietGapRow` (any blank gap ≥15 minutes)
- Response: Present `AnnotationSheet` as `.sheet`
- Minimum tap zone: Full row height (via `.contentShape(Rectangle())`)
- Affordance: "点击记录" capsule visible bottom-trailing on blank `EventCardView`; italic label on quiet gap row

### Mood Record (MAN-01)
- Entry point 1: `heart.circle.fill` button (42pt circle) in screen header — taps `viewModel.openQuickRecordComposer()`
- Entry point 2: Bottom action bar CTA ("记录此刻" / dynamic label)
- Sheet: `QuickRecordSheet` at `.medium` and `.large` detents
- Selection feedback: `withAnimation(.easeInOut(duration: 0.15))` on mood cell tap

### Manual Moment Capture (MAN-02)
- Sheet: `QuickRecordSheet` (same as mood, mode `.flexible`)
- Photo picker: `PhotosPicker`, max 3, `.images` matching
- Voice/text note: Single `TextField`, placeholder "写一句话记录当下…"
- Save path: "打点" (instantaneous) or "开始一段" (session)

### Manual Records on Timeline (MAN-03)
- Mood events render as `moodRow` — capsule shape, separate from event cards
- Mood events interleave with canvas events in `allTimelineItems` sorted array
- Mood records overlap with quiet gaps by splitting the gap into sub-segments

### History Navigation (TML-06)
- Date strip: Horizontal scroll, 30 days, auto-scrolls to selected date on appear and on change
- Calendar expand: "展开" button opens `.large` sheet with month grid
- Loading state: `ProgressView()` centered, 40pt padding
- Empty state: SF Symbol `calendar.badge.clock` at 40pt quaternary, headline + subheadline (see Copywriting)

### Recording Pipeline Status (REC-06)
- Indicator: "正在记录" / "自动记录中" / "等待首次记录" pill in `recordingStatusBar`
- Live pulse: 8pt green dot, `.easeInOut(duration: 1.5).repeatForever(autoreverses: true)` scale 1.0↔1.3, opacity 1.0↔0.6

### Active Session State
- Bottom action bar splits into two buttons: "补一个打点" (secondary) + "结束这段状态" (primary teal)
- "进行中" badge: `TodayTheme.teal` text on `TodayTheme.tealSoft` background, 11pt bold monospaced

## Animation Contracts

All animations use iOS spring system. No bouncy/elastic/playful curves. No custom spring parameters — use SwiftUI `.spring()` default unless duration precision is required.

| Animation | Curve | Duration |
|-----------|-------|----------|
| Mood cell selection | `.easeInOut` | 0.15s |
| Date strip scroll | `.default` (SwiftUI `withAnimation`) | system |
| Expand chevron rotation | implicit on `rotationEffect` | system spring |
| Sheet expand/collapse detail | `.opacity.combined(with: .move(edge: .top))` | system |
| Pulse dot (recording status) | `.easeInOut.repeatForever(autoreverses: true)` | 1.5s |
| Sheet presentation (all) | iOS default sheet spring | system |

Rule: Every animation must communicate a state change or spatial relationship. No decorative animations.

## Accessibility

- All tappable timeline rows must have `.accessibilityLabel` and `.accessibilityValue` (already on `EventCardView`)
- `DayVerticalTimelineContent` root: `.accessibilityElement(children: .contain)` + `.accessibilityLabel("今日时间轴")`
- Current-time needle: `.accessibilityHidden(!calendar.isDateInToday(timeline.date))`
- Quiet gap rows: `.accessibilityLabel("{label}，{duration}")`
- Mood grid: each cell `.accessibilityLabel("{mood.rawValue}")` + selected state `.accessibilityAddTraits(.isSelected)`
- All icon-only buttons (share, refresh, close): `.accessibilityLabel` describing action
- Minimum touch target: 44pt × 44pt enforced via `.frame(width: 44, height: 44)` on icon buttons and `.frame(maxWidth: .infinity, minHeight: 44)` on timeline rows

## Anti-Patterns — Explicitly Forbidden

These must not appear in Phase 3 code:

1. Pure `Color.gray`, `Color.white`, `Color.black` for any surface, text, or shadow — always use adaptive tokens
2. `Inter` font or any non-system typeface import
3. Purple gradients or "AI-themed" purple/blue color associations
4. Progress rings, score meters, streaks, or gamification indicators
5. Dashboard-style bar charts inside the timeline scroll area (Apple Charts used only in EventDetailView for heart rate and sleep)
6. `.spring(response:dampingFraction:blendDuration:)` with `dampingFraction < 0.7` — no bouncy physics
7. `.ultraThinMaterial` as the only background layer on colored event cards — always layer with opaque `secondarySystemGroupedBackground` first
8. Hardcoded hex strings in view files — all colors via `AppColor.*` or `TodayTheme.*` tokens
9. `TodayTheme.*` tokens in new code — new code uses `AppColor.*` directly (TodayTheme is a compat shim)
10. Non-monospaced fonts for HH:mm timestamps
