<objective>
Rewrite the onboarding flow as a multi-step sequence that: (1) shows a value screen before any permission dialog, (2) requests location as whenInUse first then upgrades to Always, (3) sequences motion after location resolves, (4) shows a recovery path when location is denied.

Purpose: Current onboarding fires both permission dialogs simultaneously and calls onComplete unconditionally regardless of outcome. This violates ONB-01/02/03 and breaks background recording silently. The two-step Always Location pattern is required by iOS 17 — calling requestAlwaysAuthorization directly on first install shows no "Always Allow" option.

Output: OnboardingView.swift rewritten as multi-step flow; project.yml usage strings updated to pass App Store Review guideline 5.1.1.
</objective>

    case value               // Show app value, no permission dialog

**LocationPermissionCoordinator** (define as a class inside the file, annotated `@MainActor`):
```swift
@MainActor
final class LocationPermissionCoordinator: NSObject, ObservableObject, CLLocationManagerDelegate {
    @Published private(set) var authorizationStatus: CLAuthorizationStatus = .notDetermined
    private let locationManager = CLLocationManager()

    override init() {
        super.init()
        locationManager.delegate = self
        authorizationStatus = locationManager.authorizationStatus
    }

    func requestWhenInUse() {
        locationManager.requestWhenInUseAuthorization()
    }

    func requestAlways() {
        locationManager.requestAlwaysAuthorization()
    }

    nonisolated func locationManagerDidChangeAuthorization(_ manager: CLLocationManager) {
        Task { @MainActor in
            self.authorizationStatus = manager.authorizationStatus
        }
    }
}
```
